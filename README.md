# Customer Churn & Revenue Intelligence Platform

An end-to-end analytics + ML product: synthetic Telco-style customer data →
MySQL → SQL analytics views → scikit-learn churn/CLV/segmentation models →
SHAP explainability → R statistical validation → Power BI dashboard → a
deployed Streamlit app. Built to demonstrate the full pipeline a data
analytics Werkstudent role expects: Python, SQL/MySQL, ML, R/statistics,
Power BI (DAX), Streamlit, and Plotly, end to end.

## Why this dataset

The data is **synthetic** (generated by `etl/01_generate_data.py`), modeled
on the structure and realistic churn drivers of the well-known IBM Telco
Customer Churn dataset — 7,043 customers, ~28.8% churn rate, with churn
probability genuinely driven by contract type, tenure, monthly charges, tech
support, payment method, and support ticket volume. This means the ML models
below are learning a real, explainable signal, not noise.

## Architecture

```
etl/           Generate + validate + load synthetic data into MySQL
  01_generate_data.py       -> data/raw/*.csv (customers, subscriptions,
                                 transactions, support_tickets, churn_events)
  02_load_to_mysql.py       -> loads + validates into churn_platform (MySQL)

sql/           Normalized schema + 10 analytics views
  01_schema.sql              customers, subscriptions, transactions,
                              support_tickets, churn_events, ml_predictions
  02_views.sql                MRR, churn rate by segment, cohort retention,
                               CLV, tenure buckets, revenue-at-risk, etc.

ml/            Machine learning layer
  01_prepare_features.py    -> pulls from MySQL, builds feature table
  02_train_churn_models.py  -> Logistic Regression / Random Forest / XGBoost,
                                 compared on ROC-AUC/precision/recall, +
                                 SHAP explainability on the winner
  03_segment_and_clv.py     -> K-Means segmentation (4 segments) +
                                 CLV regressor (predicts expected total
                                 customer lifetime in months)
  04_score_and_export.py    -> scores all customers, writes churn
                                 probability / risk tier / predicted CLV /
                                 segment back into MySQL (ml_predictions)
                                 + a static snapshot CSV for the deployed app

r_stats/       Statistical validation (base R only, no CRAN deps needed)
  churn_statistical_analysis.R  Logistic regression w/ odds ratios,
                                 chi-square tests, Welch t-tests

powerbi/       Power BI dashboard (build-it-yourself GUI steps)
  DAX_measures.txt               20+ DAX measures ready to paste in
  README_dashboard_guide.md      step-by-step 4-page dashboard build guide

streamlit_app/ Deployed interactive app
  app.py                     Overview KPIs, live churn scoring + SHAP
                              waterfall, high-risk customer explorer,
                              CLV/segmentation explorer (Plotly)
```

## Key results (this run)

- **Churn model**: Logistic Regression won (ROC-AUC 0.853), beating Random
  Forest (0.833) and XGBoost (0.847) — the underlying signal is close to
  logit-linear, so the simplest, most explainable model wins. See
  `ml/outputs/model_comparison.csv` and `roc_comparison.png`.
- **R validation** (`r_stats/output/statistical_report.txt`): McFadden's
  pseudo-R² = 0.324; contract type, tenure, tech support, and ticket count
  are all significant at p < 0.001 — confirming the same drivers SHAP
  surfaces in the ML model.
- **Revenue at risk**: ~€172K/month sits with customers in the "High" churn
  risk tier (`vw_revenue_at_risk`).
- **CLV regressor**: MAE ≈ 7.5 months, R² ≈ 0.14 — modest but real predictive
  power, which is typical/expected for CLV prediction (it's a genuinely hard
  problem; report this honestly rather than overselling it).
- **Segments**: 4 K-Means segments (High-Value Loyal, Growing/Mid-Value,
  Price-Sensitive, Low-Value/New) with distinct churn/CLV profiles.

## Reproducing this project end to end

```powershell
# 1. Python env
python -m venv .venv
.venv\Scripts\pip install -r requirements.txt

# 2. MySQL: create DB + user (edit password), then run schema
mysql -u root -p -e "CREATE DATABASE churn_platform; CREATE USER 'churn_app'@'localhost' IDENTIFIED BY 'yourpassword'; GRANT ALL ON churn_platform.* TO 'churn_app'@'localhost';"
mysql -u churn_app -p churn_platform < sql/01_schema.sql
# fill in .env with your DB credentials (see .env.example pattern in README below)

# 3. ETL
.venv\Scripts\python etl\01_generate_data.py
.venv\Scripts\python etl\02_load_to_mysql.py
mysql -u churn_app -p churn_platform < sql\02_views.sql

# 4. ML
cd ml
..\.venv\Scripts\python 01_prepare_features.py
..\.venv\Scripts\python 02_train_churn_models.py
..\.venv\Scripts\python 03_segment_and_clv.py
..\.venv\Scripts\python 04_score_and_export.py
cd ..

# 5. R stats (run from project root)
Rscript r_stats\churn_statistical_analysis.R

# 6. Run the app locally
.venv\Scripts\streamlit run streamlit_app\app.py
```

`.env` (create this yourself, it's gitignored):
```
DB_HOST=localhost
DB_PORT=3306
DB_USER=churn_app
DB_PASSWORD=yourpassword
DB_NAME=churn_platform
```
